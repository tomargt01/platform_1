Husky and Lint-Staged:

You edit a file → save.
You stage (git add) and commit.
Husky triggers → calls Lint-Staged.
Lint-Staged runs Prettier/ESLint only on the changed files.
If issues found → commit is blocked until fixed.
✅ Final result: clean, consistent, error-free code in Git.
